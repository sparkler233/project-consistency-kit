### 同步纪律(git 驱动)

事件源是 git,不手维护日志。配套 3 条纪律:

1. **B 类事件落地**:决策、对外对接、口头约定发生在对话里、git 看不见 → AI 必须**立刻写入目标文件**(决策→本文件"关键决策记录")。结束会话前建议跑一次 `/同步`,避免未落盘的 B 类事件随会话丢失。
2. **二进制按类处理**:需要版本史的栅格图走 **Git LFS** 进 git(规则见 `.gitattributes`);设计源 / PDF / Office / 字体 / 媒体 `.gitignore` 排除。**含二进制的素材 / 大文件目录**放 `_manifest.md`,只记 **用途 / 来源 / 授权或版权**(有美术 / 设计方向的项目可再加「风格基准引用」;版本归 git/LFS,不重复记);纯文档目录不要 manifest。破例纳入某被忽略二进制:先 `git lfs track "<路径>"` 再 `git add -f`(add -f 只绕过 gitignore、不走 LFS,直接加会把完整 blob 灌进 git)。〔纯代码 / 纯文字项目可简化本条,见 `.gitattributes` 说明〕
3. **草稿双轨(仅二进制)**:试稿放 `_drafts/`(已 gitignore,不进 git 也不占 LFS 额度),定稿挪上级目录 + 更新 manifest / 让 LFS 接管。

> 出向收尾用 `/同步`(git diff → 查 `一致性机制/文件联动目录.md` → 用户确认 → 落盘 → commit + 推进 `synced` tag)。入向加载用 `/catchup`。机制全貌见 `一致性机制/机制设计说明.md`;搬不动的钉死文件登记在 `一致性机制/README.md`。
