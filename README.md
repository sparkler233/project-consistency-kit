# 〔项目名〕 — 项目地图

> ⚠️ **这是【项目一致性机制】模板仓库。** 要在新项目里启用,先读 [`初始化新项目.md`](初始化新项目.md)。
> 下面是「项目地图」骨架——拷贝本仓库后,把带〔…〕的地方改写成你自己项目的内容,然后删掉本横幅。

> 本文件是项目"地图":告诉你哪里有什么、要做某件事该去哪。
> 背景、流程、决策见 [CLAUDE.md](CLAUDE.md)(项目大脑)。

## 一、顶层目录

| 位置 | 装什么 |
|------|--------|
| `CLAUDE.md` | 项目大脑——背景、流程、协作规则、关键决策 |
| `README.md` | 项目地图(本文件) |
| `一致性机制/` | 自我维护机制:catchup/同步 文档、联动规则、收尾提醒 hook |
| `.claude/` | 斜杠命令(commands/:`/catchup`、`/同步`、安装器 `/引入一致性机制`)+ hook 接线(settings.json) |
| 〔你的目录〕 | 〔正文 / 代码 / 素材 / 产物……按项目补〕 |

## 二、斜杠命令

| 命令 | 方向 | 作用 |
|------|------|------|
| `/catchup` | 入向 | 新会话开局,加载项目状态并输出报告 |
| `/同步` | 出向 | 收尾:按联动目录把改动落盘、提交、推进 `synced` |
| `/引入一致性机制` | 安装 | **跨项目全局安装器**:在别的已有项目里跑,增量合并装入本机制(不覆盖) |
