# 项目一致性机制(project-consistency-kit)

> 让 AI 协作跨会话连续工作的自我维护模板:`/catchup` 装载项目状态,`/同步` 落盘收尾,git 是唯一事件源。当前宿主为 Claude Code。

## 这是什么

AI 协作项目有两个老毛病:**新会话断片**(AI 不记得上次做到哪)和**文件间状态漂移**(README 说「待产出」,产物其实早已就位)。本仓库是一套可套用到任意项目的「一致性机制」,用 git 当唯一事件源来解决这两个问题:

- **入向 `/catchup`**:新会话开局,按四层(概览 → git 状态 → 当前焦点 → 全景扫描)把项目状态完整装载回来,输出报告后继续干活;
- **出向 `/同步`**:收尾时按「文件联动目录」把改动该联动的文档一并落盘,提交并推进 `synced` 标记;
- **兜底**:Stop hook 在有未同步改动时提醒收尾(纪律 → 制度);决策记录超 10 条自动轮转归档;
- **二进制策略**:栅格图走 Git LFS,设计源 / Office / 媒体走 gitignore + `_manifest.md` 摘要。

设计动机与取舍见 [`一致性机制/机制设计说明.md`](一致性机制/机制设计说明.md)。

## 快速上手

- **全新项目**:整包套用,步骤见 [`初始化新项目.md`](初始化新项目.md);
- **已有项目**:用全局命令 `/引入一致性机制`(装法同上文档)——增量合并装入,不覆盖现有文件,并扫描登记你项目自己的中枢文档。

## 二、斜杠命令

| 命令 | 方向 | 作用 |
|------|------|------|
| `/catchup` | 入向 | 新会话开局,加载项目状态并输出报告 |
