# 在新项目里启用「项目一致性机制」

> 这套机制让 AI 协作的项目**跨会话不走样**:新会话一条命令装载回完整状态(`/catchup`),
> 收尾一条命令把改动联动落盘、提交(`/同步`),文件之间的隐性依赖不会慢慢腐烂。
> 设计动机见 [`一致性机制/机制设计说明.md`](一致性机制/机制设计说明.md)。

## 这个模板包含什么

```
项目一致性机制/                ← 本模板仓库(拷贝它去开新项目)
├── README.md                 项目地图骨架(拷过去后改成你的)
├── CLAUDE.md                 项目大脑骨架(拷过去后填你的)
├── 初始化新项目.md            ← 本文件(套件专属,不拷入新项目)
├── CHANGELOG.md              套件版本史(套件专属,不拷入新项目)
├── .gitignore                选择性二进制排除(可按项目改)
├── .gitattributes            栅格图 LFS 规则(可按项目改 / 删)
├── .claude/commands/
│   ├── catchup.md            /catchup —— 直接可用,项目无关
│   ├── 同步.md                /同步   —— 直接可用,项目无关
│   └── 引入一致性机制.md       /引入一致性机制 —— 跨项目安装器(全局符号链接装机,不拷入新项目)
└── 一致性机制/
    ├── README.md             机制索引 + 钉死文件登记册
    ├── 机制设计说明.md         机制的"为什么"
    └── 文件联动目录.md         联动规则(通用 5 条 + 你的自定区)
```

`.claude/commands/`、`一致性机制/`、两个配置文件**直接可用**,不用改逻辑。
要填的只有两个骨架文件(`CLAUDE.md`、`README.md`)和联动目录的自定区。

> **绿地 vs 已有项目**:本指南是**绿地**路径(全新项目,整包 rsync)。要把机制装进一个**已经存在、已有自己文件 / git 历史的项目**,别用 rsync——改用全局命令 **`/引入一致性机制`**(增量 + 合并、绝不覆盖,还会扫描登记你项目自己的中枢文档)。它经全局符号链接装机(建法见下方「套件首次落机」),在那个项目里直接跑即可。

## 套件首次落机(每台机器一次)

让 `/引入一致性机制` 在本机任何项目里可用(Claude Code 的全局命令只认 `~/.claude/commands/`):

```bash
mkdir -p ~/.claude/commands
ln -sf "$HOME/Developer/项目一致性机制/.claude/commands/引入一致性机制.md" \
       ~/.claude/commands/引入一致性机制.md
```

- 用**符号链接**而非拷贝:全局命令永远跟着套件最新版,不产生第二份会过期的副本。
- 链接指向**绝对路径**:套件目录改名 / 迁移后链接即断,**需重跑上面两行**。
- `/catchup`、`/同步` 无需全局安装——它们随机制进入各项目自己的 `.claude/commands/`。

## 开张五步

**1. 把模板内容拷进你的项目**

```bash
# 在你的新项目根目录下(假设模板在 ~/Developer/项目一致性机制)
# 排除模板自己的 .git 与套件专属文件(本指南 / CHANGELOG / 全局安装器)——它们不属于新项目
rsync -a \
  --exclude='.git' \
  --exclude='初始化新项目.md' \
  --exclude='CHANGELOG.md' \
  --exclude='.claude/commands/引入一致性机制.md' \
  ~/Developer/项目一致性机制/ ./
```

**2. 填项目大脑 `CLAUDE.md`**
删掉顶部模板横幅,把所有〔…〕占位换成你项目的背景、工作流、协作分工。`同步纪律` 一节通用,原样保留(只改其中举例的项目专属部分)。

**3. 填项目地图 `README.md`**
删掉顶部模板横幅,把顶层目录表、导航表换成你项目的实际结构。

**4. 配你的领域规则与二进制策略**
- 在 `一致性机制/文件联动目录.md` 的 **Part A** 补你的中枢文档,**〔项目自定规则区〕** 补你的领域联动规则(那里有书籍项目的示范可参考)。
- 不需要图片 / 二进制版本管理的项目(如纯代码、纯文字):可清空或删掉 `.gitattributes`,并精简 `.gitignore` 里的图像 / 设计 / 媒体行。需要 LFS 的项目这一步只需**决定保留 `.gitattributes`**;实际的 `git lfs install --local` 在第 5 步 `git init` 之后执行(它要写 `.git/config`,必须先有 `.git`)。

**5. 纳入 git + 立 horizon**

```bash
git init
git branch -M main                    # 规范化分支名(老环境 git init 默认可能是 master)
git lfs install --local               # 仅 LFS 项目需要;非 LFS 项目跳过这行
git add -A
git commit -m "启用项目一致性机制:初始骨架"
git tag synced            # ★ 立 sync horizon —— /同步 从这里往后算
```

> 想要云端备份 / 版本史,再加一个 remote 并推送(GitHub/Gitee 等任选):
> `git remote add origin <你的仓库地址> && git push -u origin main && git push origin synced`

## 开张后

- 新会话开局跑 **`/catchup`** —— 装载状态。
- 干完活收尾跑 **`/同步`** —— 联动落盘 + 提交 + 推进 `synced`。
- 结束会话前建议跑一次 `/同步`,避免对话里发生的决策 / 对接没落盘就丢了。

## 升级机制本身

- **你装的是哪版**:看项目里机制文件头部的 `<!-- 一致性机制 version: YYYY-MM-DD -->` 行;**改了什么**:看套件根 `CHANGELOG.md`。要不要跟,自己决定。
- **怎么升**:别手工拷贝覆盖——在你的项目里**重跑 `/引入一致性机制`**:它按版本行判定哪些机制文件旧了,逐项问你要不要更新;你项目自己填的 `CLAUDE.md` / `README.md` / 联动目录自定区不受影响。
